Common Questions

  • I am trying to play one of your old games on modern hardware, and the game won't start, runs too slow, has blurry fullscreen, etc. Can this be fixed?

    Download the Hello Fangaming Time Capsule to get updated versions of the games that run better on modern hardware.

  • I am playing one of your old games from the Hello Fangaming Time Capsule on Windows 8.x, and the game crashes when too many sounds are playing at once. Can this be fixed?

    Unfortunately, this is a bug caused by old versions of GameMaker. Update to Windows 10 or above to fix. (Windows 7 not affected)

  • I am trying to play Super Mario Eclipse, Super Mario Dynamo or Hello Fangaming Collection, and the game is telling me there is an error defining an external function. Can this be fixed?

    Unfortunately, this is a bug caused by a DLL used by these games. Install the Microsoft Visual C++ 2010 SP1 Redistributable Package (x86) to fix.

  • I am playing Super Mario Dynamo, Mario Editor, The Shroom Project, or Shroom Editor, and the game is running too fast. Can this be fixed?

    Unfortunately, this is a bug with GameMaker: Studio 1.4. To fix this, you need to either press Alt+V to toggle VSync, or cap your frame rate to 60 using external software.

  • What are the game's controls? How do I switch to fullscreen? Etc...

    Read the manual or readme file included with each game. For the very old games that don't come with one, try pressing F1.

  • Do you recommend using GameMaker? That's what your old games used!

    No. GameMaker uses its own programming language called GML which will teach you bad programming habits. All questions related to GameMaker have been moved to Hello's Mario Engine Tutorials, and any further GameMaker related questions will be ignored.

  • Is it true that Hello Fangaming ignores criticism?

    No, this is false. Hello Fangaming has never ignored constructive criticism. When determining whether not criticism is constructive, ask yourself the following questions. Will listening to this comment improve my project? Do they have my project's best interests in mind when making the comment? If the answer to both these questions is no, then the comment is not constructive criticism, and it is actually better to ignore it.
